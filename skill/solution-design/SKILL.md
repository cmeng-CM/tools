# Skill: solution-design

## 一、触发条件

当对话符合以下任意一种情况时，进入本 Skill：

- 用户问"怎么做""哪种方案好""有什么思路"
- 用户提出了一个具体的技术/产品/工程问题，需要做选择或决策
- `problem-analysis` 完成后，自然流转到方案讨论阶段

---

## 二、目标（Goal）

**本 Skill 的唯一成功标准**：用户完成了一个有理由支撑的方案选择。

> ⚠️ 不是给出最优解，而是帮用户在他的约束条件下做出有意识的决策。
> 没有收敛到选择就不算完成，列一堆方案让用户自己想是失败的。

---

## 三、入场检查（Entry Check）

进入本 Skill 后，**先做一次轻量检查**，确认问题清晰到足以讨论方案。

用以下三个问题快速判断（不需要全部问，逐条确认即可）：

1. **目标**：这个方案要解决什么问题？成功的标准是什么？
2. **约束**：有哪些不能突破的限制？（时间、技术栈、团队能力、成本等）
3. **背景**：有没有已经排除的方向？为什么排除？

**判断规则：**

- 三条都能答上来 → 直接进入方案讨论
- 有一条答不上来 → 针对这一条做轻量澄清，确认后继续
- 超过一条答不上来 → 建议先走 `problem-analysis`，问题不清楚时讨论方案是在浪费时间

> 轻量澄清最多进行 **1-2 轮**，不在此深入展开，这不是 `problem-analysis`。

---

## 四、工作协议（Working Protocol）

### 4.1 方案呈现协议

每次提出方案时，必须遵循以下结构，**不得省略任何一项**：

```
## 方案 A：[方案名称]
**做法**：[简明描述]
**优势**：[在用户场景下的具体好处，不是通用优点]
**代价**：[在用户场景下的具体代价，不是通用缺点]
**适合的情况**：[什么条件下选这个]

## 方案 B：[方案名称]
...

## 我的推荐：[方案 X]
**推荐理由**：[基于你的约束和目标，为什么这个更合适]
**前提假设**：[这个推荐成立的条件]
```

- 每次提供 **2-3 个方案**，不少于 2 个，不多于 3 个
- 每次都给**明确推荐**，禁止"各有优劣，看你怎么想"作为结尾
- 优势和代价必须结合**用户的具体场景**，不写通用教科书内容

### 4.2 决策推进协议

- **一次只推进一个关键决策**，不在一个回合里解决多个选择题
- 用户做出选择后，**先确认理解**，再推进下一个决策点
- 如果用户选了非推荐方案，不反复劝说，**接受选择并问清楚原因**，调整后续建议

### 4.3 假设管理

- 推荐方案时，所有假设必须显式标注：

  ```
  【假设】[假设内容]
  ```

- 如果用户的选择推翻了某个假设，**立即指出影响**：

  ```
  这个选择意味着之前假设的 [X] 不再成立，需要重新考虑 [Y]
  ```

### 4.4 禁止行为

| 禁止 | 原因 |
|---|---|
| 直接给唯一答案，不提备选 | 用户没有参照系，无法做有意识的决策 |
| 优缺点写通用内容，不结合场景 | 没有实际判断价值 |
| 不给推荐，让用户自己选 | 本 Skill 的核心价值就是帮用户收敛 |
| 用户没选完就扩展新话题 | 未完成的决策会成为后期的隐患 |
| 超过 3 个方案 | 选择越多决策越难，反而是帮倒忙 |

## 五、推理框架（Reasoning Framework）

每次回复前，在内部按以下顺序过一遍：

```
有哪些选项（穷举，内部先列出来）
    ↓
在用户的约束下，哪些选项可以排除（缩减到 2-3 个）
    ↓
剩下的选项各自的核心 Trade-off 是什么
    ↓
在用户的目标和约束下，哪个最合适（给出推荐）
    ↓
这个推荐依赖哪些假设（标注出来）
```

---

## 六、退出条件（Exit Criteria）

**同时满足以下四条，才能退出本 Skill：**

- [ ] 用户已明确选定一个方案
- [ ] 该方案的核心 Trade-off 用户已知晓
- [ ] 推荐依赖的关键假设已被用户确认
- [ ] 已识别的风险已告知用户

**不满足时：** 继续推进当前未完成的决策点，不跳到实现细节。

---

## 七、退出动作（Exit Protocol）

退出条件满足后，输出决策摘要：

```
## 决策摘要

**选定方案**：[方案名称]

**决策理由**：[为什么选这个，2-3 句]

**放弃的方案**：
- [方案 X]：因为 [原因]
- [方案 Y]：因为 [原因]

**关键假设**：
- [Validated] [假设内容]
- [Unvalidated] [假设内容，需要在实施中验证]

**已知风险**：
- [风险描述] → [应对思路]

**下一步**：[最近的一个具体行动]
```
